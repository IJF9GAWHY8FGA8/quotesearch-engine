# QuoteSearch Engine

QuoteSearch Engine is a Python command-line search tool that crawls
`https://quotes.toscrape.com/`, builds an inverted index of the collected quote
pages, and lets users search for single words or multi-word queries.

The project was built for COMP/XJCO3011 Coursework 2: Search Engine Tool.

## Features

- Crawls the required target site: `https://quotes.toscrape.com/`
- Respects a six-second politeness window between consecutive requests
- Handles request failures without an uncontrolled crash
- Builds a JSON inverted index with:
  - term frequency per page
  - word positions per page
  - document frequency
  - page metadata
- Supports the required CLI commands:
  - `build`
  - `load`
  - `print`
  - `find`
- Supports case-insensitive single-word and multi-word search
- Handles empty queries, missing terms, and special-character input
- Includes unit and CLI tests with coverage reporting

## Installation

Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

The project uses:

- `requests` for HTTP requests
- `beautifulsoup4` for HTML parsing
- `pytest` and `pytest-cov` for testing

## Usage

Run commands from the repository root.

### Build

```bash
python3 src/main.py build
```

This crawls the quote listing pages, builds the inverted index, and saves it to:

```text
data/index.json
```

The build command intentionally takes about one minute because it waits at least
six seconds between consecutive requests to the target site.

### Load

```bash
python3 src/main.py load
```

This loads the saved compiled index and prints a summary.

### Print

```bash
python3 src/main.py print nonsense
```

This prints the inverted index entry for the normalized term, including document
frequency, page URLs, frequency, and positions.

### Find

```bash
python3 src/main.py find indifference
python3 src/main.py find good friends
```

`find` returns pages that contain all query terms. Results are ranked by:

1. exact phrase match, when query terms appear in consecutive positions
2. total query-term frequency
3. URL order as a deterministic fallback

## Inverted Index Design

The compiled index is stored as JSON:

```json
{
  "meta": {
    "version": 1,
    "base_url": "https://quotes.toscrape.com/",
    "page_count": 10,
    "total_tokens": 3436,
    "unique_terms": 858
  },
  "pages": {
    "https://quotes.toscrape.com/page/2/": {
      "title": "Quotes to Scrape",
      "word_count": 430
    }
  },
  "index": {
    "good": {
      "doc_freq": 6,
      "postings": {
        "https://quotes.toscrape.com/page/2/": {
          "frequency": 3,
          "positions": [8, 128, 313]
        }
      }
    }
  }
}
```

This structure is suitable for the coursework because it is simple to inspect in
the video, easy to save and load, and supports both frequency-based ranking and
phrase checks using word positions.

## Design Decisions

- The crawler follows the quote listing pagination links. This covers the main
  relevant quote pages while keeping the six-second politeness requirement
  practical for demonstration and marking.
- Tokenization is case-insensitive, so `Good`, `GOOD`, and `good` are indexed as
  the same term.
- Word positions are stored because they allow phrase-aware ranking for
  multi-word queries.
- The index is saved as one JSON file because the target site is small and the
  coursework allows a single compiled index file.
- Tests use mocks for network and sleep behavior. Real network crawling is still
  verified manually through `python3 src/main.py build`.

## Edge Cases Covered

- Network request failure during crawling
- Missing index file
- Invalid JSON index file
- Empty query
- Query containing only special characters
- Missing search terms
- Mixed-case input
- Multi-word query handling
